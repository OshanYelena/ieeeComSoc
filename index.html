Spectrum Sharing and Spectrum Misuse

Challenge

  • Radio spectrum is limited and becoming expensive due to the ever-growing demand for mobile network services
  • Dynamic Spectrum Access (DSA) is an viable solutions to mitigate scarcity of spectrum
    • Co-Primary Spectrum Sharing (CoPSS), Cognitive Radio (CR), Spectrum Access System (SAS),
    • Must needed for Local Private Mobile Networks/Micro-operators
  • Malicious user can violate the spectrum sharing agreements and also transmit on illegal spectrums
  • No real-time spectrum misuse detection system

Goal

Develop a customized blockchain consensus algorithm for Dynamic Spectrum Access (DSA) and detect spectrum misuse

Blockchain for DSA

  • Blockchain is a viable option to enable DSA
  • Suffering from several challenges

Spectrum Sensor

HackRF SDR Capabilities

  • 1 MHz to 6 GHz operating frequency Half-duplex transceiver
  • Compatible with GNU Radio
  • Up to 20 million samples per second
  • Software-configurable RX and TX gain and baseband filter
  • Software-controlled antenna port power (50 mA at 3.3 V)

Raspberry Pi benefits

  • Act as a interface to interact with HackRF
  • Can program the channels
  • Can used to process the data and find the winning node in the network

Implementation of Hardware Platform :

Spectrum Sensors Architecture

AI based user identification & Automatic Misuse detection

  • Identify the users with radio signature
    • To stop malicious transmitter from impersonation
    • To mitigate RF interference
  • Automatic smart contract
    • Detect the malicious activity
    • Analyze existing smart contract
    • Identify what set of data uploaded
